Why This System Scores Well

Base-rate legal QA failures are predictable: wrong source, unsupported aggregation across laws, format drift, and latency spikes on the hardest free-text cases. The pipeline is designed against those failure modes directly.

Common failure pattern
Wrong source, wrong shape, too much confidence
Most systems lose points by missing the governing document, over-speaking beyond the evidence, or drifting from the required answer type.
Design response
Constrain the answer path before the answer is trusted
Retrieval discipline, answer-type routing, final-answer cleanup, and citation provenance checks are applied to reduce exactly those failure modes.

Retrieval discipline

Dense search, sparse search, and reranking are used to shrink a large candidate set into a small context window with a clear evidence trail.

Answer-type control

Strict answer types stay short and typed. Multi-document free-text answers are routed differently from deterministic booleans, names, dates, and numbers.

Final-answer provenance

Answer cleanup and citation checks are applied before the final response is presented.

24-hour reliability

Circuit breakers, backoff, fallback chains, and Dockerized evaluation reduce the risk that a good system collapses in unattended operation.

Method Appendix

Implementation detail kept compact: enough to assess rigor, reproducibility, and control flow.

Retrieval path

Hybrid dense + sparse retrieval is narrowed by reranking before generation, so the model sees a small, evidence-rich context window rather than a broad uncurated dump.

Answer routing

Strict answer types stay on short typed paths; broader multi-document reasoning can upgrade to slower but more reliable generation paths when needed.

Verification and telemetry

Final-answer cleanup, citation validation, and per-stage telemetry are attached to the emitted answer so debugging and judging happen on the same artifact.

Reproducibility

The repository, Dockerized evaluation flow, and answer-level telemetry are available so the strongest claims on this page can be re-run rather than merely admired.

Stress Tests and Limits

The strongest status signal is not pretending the system is magic. It is showing where pressure still exists and what evidence would change our confidence.

Long-tail latency still matters

Latency is already in a strong range on the verified local run. The competition factor averages per-answer TTFT modifiers, so hidden grounding still matters more than shaving the median path.

Multi-law reasoning is the brittle edge

The most fragile class is cross-law transfer: letting one commencement or enactment fact bleed into another law. That is watched more aggressively than generic single-law questions.

Claims are bounded by current evidence

This page does not make claims about the private set. It shows why the public-trial evidence already deserves serious inspection.
